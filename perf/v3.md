# SIMD v3 Accepted Optimisation

This checkpoint records the accepted `v3` optimisation for `src-simd2`.

## Change Summary

Accepted optimisation:

- item `6` from [`perf_plan_cx.md`](/home/lloydf/zigraster/perf_plan_cx.md):
  resolve only touched output pixels

What changed:

- kept the existing full scratch clears
- added per-subpixel-row touched x bounds during rasterization
- updated touched bounds only when a subpixel write passed depth and committed
- used those bounds in `averageScratch` to skip untouched output rows and x ranges
- added an explicit `sub_sample == 1` bypass around the averaging resolve call
- specialized `averageScratch` for `fields_num == 1` and `fields_num == 3`
- kept a generic `else` path for other field counts

Why this worked:

- it reduces resolve work without adding the more invasive dirty-region tracking from
  rejected item `5`
- it benefits all kernels that write into scratch and then use the shared resolve path,
  including the nonlinear Newton-family kernels

## Performance Summary

Overall suite gains versus `v2`:

- `fullraster`: `+18.37%`
- `geom`: `+9.03%`
- `sphere2000`: `+14.11%`

Low-order triangle gains:

- `tri3`
  - `fullraster`: `+32.40%`
  - `geom`: `+12.64%`
  - `sphere2000`: `+18.92%`
- `tri3opt`
  - `fullraster`: `+20.04%`
  - `geom`: `+10.81%`
  - `sphere2000`: `+18.11%`

Newton-family gains:

- overall average across the three benchmark suites:
  - `quad4newton`: `+13.12%`
  - `tri6`: `+9.56%`
  - `quad8`: `+10.83%`
  - `quad9`: `+10.14%`

Representative wins:

- `fullraster:tri3_flat_grey +139.86%`
- `fullraster:tri3opt_flat_grey +114.78%`
- `fullraster:tri3_flat_rgb +72.18%`
- `fullraster:tri3opt_flat_rgb +51.64%`
- `sphere2000:quad9_tex8_rgb +38.79%`
- `fullraster:quad4newton_flat_grey +33.98%`

Regression check:

- no non-`tri3` or non-`tri3opt` regressions worse than `-1%` were observed in the
  acceptance run

## Status

`v3` is accepted and should be treated as the current best known `src-simd2`
checkpoint until a better `v4` candidate is established.
