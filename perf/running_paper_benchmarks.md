# Running Paper Benchmarks

This guide explains how to execute the benchmarking suite using the provided Python orchestration scripts.

## 1. Prerequisites
Before running the Python scripts, you must compile the benchmark binaries. The scripts expect both `scalar` and `simd` versions of the executables in the `./bin/` directory.

```bash
# Compile all benchmark binaries (creates ./bin/bench_*)
./scripts/compile_benchmarks.sh
```

## 2. Running General Benchmarks
The general benchmarks (`fullraster`, `geom`, `sphere2000`, `sphere2000zoom`) use a consistent interface.

```bash
# Run the full raster benchmark with default settings (all experiments)
python3 ./scripts/bench_fullraster.py

# Run only a specific experiment (e.g., Experiment 1: SIMD vs Scalar)
python3 ./scripts/bench_fullraster.py --experiment 1

# Specify the number of timing runs per case
python3 ./scripts/bench_geom.py --runs 5

# Perform a dry run to see the commands that would be executed
python3 ./scripts/bench_sphere2000.py --dry-run
```

## 3. Running the DIC UQ Benchmark
The `bench_dicuq.py` script covers more advanced threading and pipeline experiments.

```bash
# Run all DIC UQ scaling and threading experiments
python3 ./scripts/bench_dicuq.py

# Run a specific experiment (e.g., Experiment 3: Frames in Flight)
python3 ./scripts/bench_dicuq.py --experiment 3
```

## 4. Output and Results
By default, results are written to:
`./out/benchmark_runs/<benchmark_name>/<timestamp>/`

Inside this directory, you will find:
*   **`experiment_X/`**: Subdirectories for each experiment matrix.
*   **`stdout.txt` / `stderr.txt`**: Raw output from the Zig binaries.
*   **`command.txt`**: The exact shell command used for that specific case.
*   **`benchmark_report.txt`**: A formatted table of timing results and statistics (generated by the Zig binary).

# Default Configurations

When run without arguments, the scripts use a hierarchy of defaults. The Python orchestration layer sets viewport and thread defaults, while the Zig layer handles internal execution defaults like the number of runs.

## 1. Python Orchestration Defaults
These are applied to all general benchmarks (`fullraster`, `geom`, `sphere2000`, `sphere2000zoom`).

| Parameter | Default Value | CLI Argument |
| :--- | :--- | :--- |
| **Viewport Width** | `1600` pixels | `--pixels-x` |
| **Viewport Height** | `1000` pixels | `--pixels-y` |
| **Sub-sampling** | `1` (1x1) | `--sub-sample` |
| **Total Threads** | `1` | `--total-threads` |
| **Experiment Filter** | `"all"` | `--experiment` |
| **Runs** | `10` (from Zig) | `--runs` |

## 2. DIC UQ Benchmark Defaults
Specific defaults used by `scripts/bench_dicuq.py` for its experiments.

| Parameter | Default Value | Description |
| :--- | :--- | :--- |
| **Texture Sample** | `cubic_catmull_rom` | High-quality cubic interpolation |
| **Sample Mode** | `lut_lerp` | Uses Look-Up Tables with linear interpolation |
| **Frames in Flight** | `1` | Number of frames processed concurrently |
| **Render Mode** | `offline` | Batch processing mode |

## 3. Internal Zig Defaults
If a parameter is not explicitly passed by the Python script, the underlying Zig binary uses these values.

| Parameter | Default Value |
| :--- | :--- |
| **Runs** | `10` |
| **Viewport** | `800x500` (Overridden by Python default of 1600x1000) |
| **Sub-sampling** | `2` (Overridden by Python default of 1) |
